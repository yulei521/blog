学习hexo搭建站点所遇到问题小结

发表于 2016-01-20   |   分类于 hexo   |     |   阅读次数

hexo是一个基于node.js的快速、简洁且高效的博客框架，使用Markdown来编写文章。

hexo 安装

首先确保本地已经安装好了node.js，git,不同平台操作方法不同。具体可参考hexo官方文档，里面详细介绍了安装流程。下面主要说下我遇到的一些问题，方便大家参考。

运行 hexo generate 指令所带来的问题

在hexo指令中，hexo generate指令是生成静态文件。当运行这条指令时，就会重新生成/public目录下的文件比如.html文件，css文件等，如果你只在/public目录下新添加了一个css文件，那么在下一次运行hexo generate指令时，会自动删除该的文件。

那么我该怎么添加或者修改文件呢？

如果是修改主题(themes)相关的文件,需要在themes文件中找到对应的源文件进行修改，这里的html文件都是.ejs格式的，存放目录是/themes/icarus(我主题的名字)/layout；/themes/icarus(我主题的名字)/source存放的是css和js文件，找到相应的部分进行修改即可，添加的话也是添加到相应的文件夹下面。然后再运行 hexo generate 指令，在/public目录生成的文件就是修改后的文件了。

如果是新加的js文件或者css文件，可以在/soucre目录下新建文件夹js或者css,将新建的文件放到对应的目录下，然后运行hexo generate指令，新加的文件才会在/public目录下显示。
比如：
我在/soucre目录下新建了一个css文件夹，然后在css文件夹下新建了一个new.css。

运行指令：

hexo genreate

然后在/public/css目录下面可以看到刚才新建的css文件new.css

切记：不要直接在/public目录下直接添加文件。

如何自定义页面

命令行运行新建页面指令

hexo new page “about”

在/source目录下会生成about目录，里面有个index.md文件，直接编辑就可以了，然后记得在主题的_config.yml中将其配置显示出来

如何将hexo项目发布到github

hexo提供了一键发布到git指令，即hexo deploy,前提是你已经注册了github账号，并已经配置好了目录下的_ config.yml文件（不是主题下的_config.yml）。
注册githun账号，直接到github官网注册就行，然后创建一个新的库

如何域名绑定

新买了一个域名，如何绑定，首先我们需要把项目发布成github pages，然后才可以进行域名绑定，具体可参考官网, 里面详细介绍了什么是GitHub Pages以及如何发布，域名绑定。
域名绑定这里还有个中文文档可以参考：http://jingyan.baidu.com/article/dca1fa6fa1e403f1a5405262.html

win10会导致Markdown2预览不正常

下载个插件Awesomium 1.6.6 SDK，安装好重启就可以了。

其他问题

禁用评论，在文章中加上如下字段

comments: false

description 字段，在不同的主题下意义不同，主要是对文章的描述，当添加”阅读全文”，会显示description字段的值。
